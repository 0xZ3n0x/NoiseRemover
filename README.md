# NoiseRemover

**Speech Enhancement in Noisy Environments Using U-Net Based Audio Denoising**

A deep learning system that removes background noise from speech recordings. Takes a noisy audio file as input and outputs a clean, denoised version.

---

## How It Works

The model is trained on synthetic noisy-clean pairs generated by mixing LibriSpeech (clean speech) with ESC-50 environmental noise clips at multiple SNR levels. A U-Net encoder-decoder predicts a noise suppression mask applied to the Mel spectrogram of the input audio. The enhanced spectrogram is converted back to a waveform via inverse STFT.

```
Noisy audio → STFT → Mel spectrogram → U-Net mask → iSTFT → Clean audio
```

### Signal Processing Background

#### Short-Time Fourier Transform (STFT)

A raw waveform $x[n]$ is cut into short overlapping frames and a Fourier transform is applied to each:

$$X[m, k] = \sum_{n=0}^{N-1} x[n + mH] \; w[n] \; e^{-j2\pi kn/N}$$

| Symbol | Meaning | Value used |
|--------|---------|------------|
| $N$ | FFT size | 1024 |
| $H$ | hop length (frame step) | 256 samples |
| $w[n]$ | Hann analysis window | — |
| $m$ | frame index (time axis) | — |
| $k$ | frequency bin index | $0 \ldots N/2$ |

The result is a complex matrix. Its magnitude $|X[m,k]|$ is the spectrogram; its phase $\angle X[m,k]$ is kept separately for reconstruction.

---

#### Mel Spectrogram

The linear frequency axis ($N/2 + 1 = 513$ bins) is projected onto the **Mel scale** — a perceptual frequency scale that mirrors how the cochlea resolves pitch:

$$f_\text{mel} = 2595 \cdot \log_{10}\left(1 + \frac{f}{700}\right)$$

A triangular filterbank matrix $\mathbf{M} \in \mathbb{R}^{128 \times 513}$ sums energy within each Mel band:

$$S_\text{mel} = \mathbf{M} \cdot |X|$$

Lower frequencies are resolved finely; higher frequencies are grouped into wider bands — matching human auditory sensitivity.

---

#### Decibel (dB) Conversion

Mel amplitudes span several orders of magnitude. Converting to decibels compresses this into a linear, numerically tractable range:

$$S_\text{dB} = 20 \cdot \log_{10}(S_\text{mel} + \epsilon)$$

This maps the ~120 dB physical dynamic range of audio to a compact domain, so the model's MSE loss is balanced between quiet and loud regions.

---

#### Noise Mask and Enhancement

The model predicts a soft mask $M \in [0, 1]^{128 \times T}$ over the spectrogram. Each value $M_{m,k}$ controls how much of the corresponding time-frequency bin is retained.

The mask is applied in the **linear amplitude domain** (not dB), because multiplicative suppression is physically correct there:

$$A_\text{noisy} = 10^{\,S_\text{noisy}\,/\,20}$$

$$\hat{S} = 20 \cdot \log_{10}(A_\text{noisy} \cdot M + \epsilon)$$

- $M_{m,k} = 0$ → complete suppression (silence)
- $M_{m,k} = 1$ → no change (original signal preserved)

---

#### Inverse STFT (iSTFT)

To reconstruct a waveform the model's enhanced magnitude is combined with the **original noisy phase**:

$$\hat{x}[n] = \text{iSTFT}\left(\hat{A}[m,k]\cdot e^{\,j\angle X_\text{noisy}[m,k]}\right)$$

Recovering phase from magnitude alone is an ill-posed problem. Reusing the noisy phase is standard practice; its perceptual impact on speech quality is negligible.

---

#### Evaluation Metrics

**SNR (Signal-to-Noise Ratio)** — ratio of speech power to noise power:

$$\text{SNR} = 10 \cdot \log_{10} \frac{\|s\|^2}{\|n\|^2} \quad [\text{dB}]$$

**SNRi (SNR Improvement)** — how much the model raised the SNR, independent of starting conditions:

$$\text{SNRi} = \text{SNR}_\text{output} - \text{SNR}_\text{input}$$

**PESQ (Perceptual Evaluation of Speech Quality)** — ITU-T P.862 standard that models human listener scores (MOS). Computed by comparing the enhanced waveform against the clean reference through a perceptual filterbank. Score range: 1 (bad) → 4.5 (excellent).

## Usage

### Denoise a file

```bash
# From a test sample (.npz)
python scripts/infer.py sample

# From a full audio file mixed with ESC-50 noise
python scripts/infer.py long --noise rain --snr 0
python scripts/infer.py long --noise siren --snr 5

# Use GPU
python scripts/infer.py sample --device cuda
```

### Interactive demo

```bash
python scripts/demo.py          # CPU (default)
python scripts/demo.py --device cuda
python scripts/demo.py --share  # public Gradio link
```

### Training from scratch

```bash
# Download datasets
bash scripts/download_data.sh

# Prepare spectrogram pairs
python scripts/prepare_data.py \
    --config configs/default.yaml \
    --max-files-per-speaker 5 \
    --max-chunks-per-file 5

# Train
python scripts/train.py --config configs/default.yaml --run-name baseline

# Evaluate
python scripts/evaluate.py --config configs/default.yaml --checkpoint checkpoints/best.pt
```

## Model

U-Net encoder-decoder with ~31M parameters.

```
Input:       (B, 1, 128, T)  — log-mel spectrogram
Encoder:     1 → 32 → 64 → 128 → 256 → 512
Bottleneck:  512 → 1024  (Dropout 0.3)
Decoder:     1024 → 512 → 256 → 128 → 64 → 32  (Dropout 0.2)
Output:      (B, 1, 128, T)  — mask in [0, 1]
```

The mask is applied in linear amplitude domain: `enhanced = amplitude × mask`. This ensures mask=0 produces silence and mask=1 preserves the original signal.

## Results

Trained on ~30k samples (LibriSpeech + ESC-50, 5 SNR levels: -5 to 15 dB, 251 speakers, 50 noise categories).

| Metric | Value |
|--------|-------|
| SNRi (mean) | 8.49 dB |
| PESQ (mean) | 2.84 |

**Per SNR level:**

| Input SNR | SNRi | PESQ |
|-----------|------|------|
| -5 dB | 8.34 dB | 2.10 |
| 0 dB | 11.30 dB | 2.52 |
| 5 dB | 10.42 dB | 2.89 |
| 10 dB | 7.91 dB | 3.27 |
| 15 dB | 4.47 dB | 3.41 |

**Inference speed (CPU):** 120.7 ms per 4-second chunk → 13.8x realtime
